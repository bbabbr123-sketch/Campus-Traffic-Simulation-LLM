# Campus Traffic Simulation & Dynamic Scheduling Engine

基于多智能体仿真、人工势场与大语言模型辅助决策的校园共享单车调度系统。

本项目尝试针对校园共享单车在不同时段、不同区域出现的供需不平衡问题，构建一个微观交通仿真环境，并在此基础上探索路径规划、动态调度、状态预测与 LLM 辅助决策方法。

## 项目背景

校园共享单车在早晚高峰、教学楼与宿舍区之间常出现明显的时空供需不平衡：部分区域车辆堆积，而部分区域无车可用。本项目希望通过仿真方式复现这一现象，并尝试设计动态调度策略，提高系统整体运行效率。

## 核心功能

### 1. 多智能体微观仿真

- 构建离散网格环境，模拟用户、车辆、站点与区域需求变化
- 设计用户出行、车辆流动、区域供需缺口等状态变量
- 支持无调度与有调度两种模式下的对比实验

### 2. 路径规划与拥堵处理

- 使用 BFS 进行基础路径搜索与距离估计
- 引入人工势场思想，模拟个体之间的空间排斥与局部避让
- 使用 Softmax 概率决策机制缓解高密度场景下的局部死锁问题
- 在对冲场景中加入方向偏置项，尝试模拟靠右通行规则

### 3. 供需预测与动态调度

- 基于历史状态数据构建时空特征
- 使用 MLP 模型预测区域供需缺口
- 根据预测结果生成调度建议，并评估调度前后的系统指标变化

### 4. LLM 辅助调度探索

- 接入大语言模型 API，将自然语言调度建议转化为系统可执行指令
- 在执行前通过沙盒仿真进行短期推演，检查调度指令是否可能加剧局部供需失衡
- 探索 LLM 在复杂系统宏观决策中的辅助作用与局限性
